# Termo Unificado: Responsabilidade e Conformidade

**Projeto:** Smart Attendance
**Instituição:** CEUB – Centro Universitário de Brasília

## 1. Finalidade e Contexto
Este documento unifica as políticas de uso deste protótipo. O sistema é uma prova de conceito acadêmica para validar a eficiência da chamada automatizada, operando exclusivamente em ambiente local para fins de demonstração e avaliação pela docência.

* **Objetivo:** Simulação funcional para validação de requisitos e lógica de negócio.
* **Base Legal:** Tratamento baseado no **Interesse Legítimo** para fins estritamente acadêmicos e pedagógicos.

## 2. Coleta e Natureza dos Dados (Simulação)

* **Dados Não-Sensíveis:** Os usuários (alunos) devem inserir apenas dados fictícios ou genéricos.
* **Ausência de Banco de Dados:** Não há persistência permanente ou banco de dados real. Os dados existem apenas durante a execução da instância do protótipo (em memória ou arquivos temporários locais), eliminando riscos de vazamento em larga escala.
* **Anonimização:** Como a entrada de dados já é descaracterizada pelo próprio usuário, não são necessários processos de anonimização no sistema.

## 3. Infraestrutura e Visibilidade
Devido à natureza didática e à ausência de dados reais sensíveis:

* **Repositório Público:** O código-fonte é mantido em repositório **Público** para permitir a auditoria e avaliação pela equipe docente.
* **Sem Comunicação em Rede:** O protótipo é uma aplicação local *standalone*. Não há APIs externas, protocolos de comunicação complexos ou tráfego de dados via rede que exijam criptografia.
* **Segurança:** A segurança baseia-se na não-coleta de dados reais e na execução local controlada.

## 4. Responsabilidades de Uso
* **Conteúdo:** É vedada a inserção de qualquer dado pessoal real, ofensivo ou sensível nos campos de teste do protótipo.
